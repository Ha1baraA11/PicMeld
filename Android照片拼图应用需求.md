# Android 照片拼图应用 — 最终确认版需求

## 一、应用定位

- 平台：Android 原生应用
- 目标设备：小米手机
- 目标系统：Android 8.0 及以上（重点兼容 8.0 / 9.0）
- 使用方式：自己使用
- 网络要求：纯离线，不接入云端

## 二、核心功能

- 仅支持从系统相册选择图片
- 支持一次选择最多 200 张
- 不允许重复选择同一张图片
- 选中的图片以缩略图网格形式展示（每行 3 张）
- 支持从已选列表中删除单张图片（右上角删除按钮）
- 重新选择图片时采用追加模式

## 三、拼图规则

- 每 4 张图片为一组
- 每组输出 1 张 2×2 拼图
- 图片摆放顺序：左上 → 右上 → 左下 → 右下
- 若最后一组不足 4 张：空白位置使用纯白色填补
- 拼图之间无间距
- 留白颜色：白色
- 图片处理方式：等比缩放、完整显示、允许留白、不裁剪

## 四、图片特征与处理

- 用户场景默认以截图为主
- 默认认为四张图尺寸通常接近
- 自动读取并修正图片方向（EXIF）
- 若某张图片读取失败、损坏或格式异常：终止任务，弹窗提示错误

## 五、输出规则

- 输出格式：JPG
- 保存方式：直接保存到系统相册，不单独创建相册目录
- 生成完成后：提示"完成"，自动清空已选列表

## 六、进度与交互

- 显示进度条
- 进度计算：已处理图片数 / 总图片数，每处理完 1 张图片更新一次
- 建议同时显示百分比和"已处理张数 / 总张数"
- 处理中：禁用"选择图片"和"开始生成"按钮
- 允许切到后台，但以前台处理最稳定为目标

## 七、界面要求

- 界面语言：中文
- 风格要求：简洁易用
- 页面结构：单页极简结构
- 包含：标题、已选数量、预计生成张数、缩略图列表、选择图片按钮、开始生成按钮、进度条与进度文本

## 八、实现细节

- minSdk = 26
- 开发语言：Kotlin
- 开发工具：Android Studio
- UI 方案：原生 XML + ViewModel
- 图片选择：兼容方案，确保 Android 8/9 可用
- 缩略图布局：每行 3 张
- 删除方式：缩略图右上角删除按钮
- 完成用简短提示，错误用弹窗提示

## 九、预计生成张数计算

预计生成张数 = ⌈总图片数 / 4⌉

示例：
- 1~4 张 → 1 张拼图
- 5~8 张 → 2 张拼图
- 9~12 张 → 3 张拼图
