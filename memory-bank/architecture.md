# 架构与拓扑 (Architecture)

**核心设计**: Android 原生单页架构 (Single-Page Arch)

## 模块划分与数据流向
1. **UI 视图层 (View)**: 
   - 基于原生 XML 约束布局或线性布局。
   - 包含文件选择、功能操作按钮、RecyclerView 缩略图展示以及处理进度条指示。
2. **逻辑状态层 (ViewModel)**: 
   - 管理当前已选图片列表、最大上限(200张) 检查。
   - 提供自动计算预计生成拼图数量 (⌈N / 4⌉)。
3. **图像处理管道 (Image Processing Pipeline)**:
   - **读取模块**: 访问系统相册 URI 集合，并结合 EXIF 自动修正原图像方向。
   - **拼接模块**: 处理输入序列，每 4 张切分为一组。不足 4 张时使用白色背景填充空位。执行等比缩放 (保持完整显示、不裁剪)。
   - **输出模块**: 编码为 JPG 格式，回踩直接存入用户系统相册。
